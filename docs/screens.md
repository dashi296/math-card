# 画面仕様書

## 概要

Math Card アプリの画面構成は、Expo Router によるファイルベースルーティングと Feature-Sliced Design に基づいています。

---

## 画面一覧

| 画面名 | ルート / ファイル | 種別 |
|---|---|---|
| ホーム画面 | `app/(tabs)/index.tsx` | タブ画面 |
| カードセット選択画面 | `features/card-sets/ui/card-set-selector.tsx` | 組み込みビュー |
| カードセット練習画面 | `features/card-sets/ui/card-set-flashcard.tsx` | 組み込みビュー |
| 完了画面 | `features/card-sets/ui/card-set-flashcard.tsx` (内部) | 組み込みビュー |
| Explore 画面 | `app/(tabs)/explore.tsx` | タブ画面 |
| モーダル画面 | `app/modal.tsx` | モーダル |
| 強制アップデートモーダル | `features/force-update/ui/force-update-modal.tsx` | オーバーレイモーダル |

## 1. ホーム画面

**ルート**: `app/(tabs)/index.tsx`

### 概要

アプリのメイン画面。ヘッダーとカードセット練習コンポーネントを表示する。

### UI 構成

- **ヘッダー**
  - 絵文字アイコン (🧮)
  - タイトル: 「音声で計算練習」
  - サブタイトル: 「声で答えて、どんどん上達しよう」
- **フラッシュカード領域**
  - `CardSetFlashcard` コンポーネントを埋め込み表示
  - カードセットが未選択の場合はカードセット選択画面を表示

### 状態遷移

```
ホーム画面
  └── CardSetFlashcard (カードセット未選択)
        └── カードセット選択画面 → カードセット練習画面 → 完了画面
```

## 2. カードセット選択画面

**コンポーネント**: `features/card-sets/ui/card-set-selector.tsx`

### 概要

練習するカードセットを選択する画面。DB から読み込んだカードセット一覧を表示し、未登録の場合はデフォルトカードセット（1・2年生分）を自動初期化する。

### UI 構成

- **タイトル**: 「カードセットを選んでね」
- **サブタイトル**: 「練習したいセットをタップしてください」
- **カードセット一覧** (スクロール可能)
  - 各カードアイテム:
    - 演算子バッジ (絵文字: ➕ ➖ ✖️ ➗)
    - カードセット名
    - 詳細情報: 演算の種類・答えの範囲・枚数
    - 選択ボタン (▶)
    - 選択中は背景色・ボーダー色を変更

### 状態

| 状態 | 表示 |
|---|---|
| 読み込み中 | 「カードセットを読み込み中...」 |
| カードセットなし | エラーメッセージ + 「再読み込み」ボタン |
| 正常 | カードセット一覧 |

### データ

- DB から全カードセットを取得 (`getAllCardSets`)
- 初回起動時に `GRADE1_CARD_SETS` / `GRADE2_CARD_SETS` を自動登録

## 3. カードセット練習画面

**コンポーネント**: `features/card-sets/ui/card-set-flashcard.tsx`

### 概要

選択したカードセットの問題を音声認識で練習する画面。問題表示・音声入力・採点・進捗管理を行う。

### UI 構成

- **ヘッダー**
  - カードセット名
  - 進捗テキスト: 「N / M 枚目」
  - プログレスバー

- **統計エリア**
  - 正解数 / 不正解数 / 正解率（%）を横並び表示

- **問題カード**
  - 計算式を大きく表示（例: `3 + 5 = ?`）
  - 演算子は `*` を `×` に変換して表示

- **音声認識ステータスエリア**
  - 聞き取り中: 「🎤 聞き取り中...」+ ヒント文
  - 認識中（暫定）: 認識中のテキストをリアルタイム表示

- **フィードバックカード**
  - 正解: 「🎉 正解!」(緑背景)
  - 不正解: 「😅 残念!」(赤背景) + ユーザーの答えと正解を表示

- **エラーカード** (音声認識エラー時)

- **アクションボタン**
  - 未開始: 「🎤 開始する」(緑)
  - 聞き取り中: 「⏸ 一時停止」(赤)
  - 再認識可能時: 「🎤 再認識」(黄)
  - 不正解後: 「🔄 もう一度挑戦」(黄)
  - 「← カードセット選択に戻る」(テキストボタン)

- **使い方説明** (画面下部)
  1. 「開始する」ボタンを押す
  2. 計算の答えを声で言う（例：「じゅうご」）
  3. 自動で採点
  4. 正解 → 次の問題へ自動進行
  5. 不正解 → 正解するまで同じ問題を繰り返す

- **DEV 専用** (開発環境のみ): 「[DEV] 最後の1問まで飛ばす」ボタン

### 音声認識フロー

```
開始ボタン押下
  └── 音声認識開始 (expo-speech-recognition, ja-JP)
        ├── 暫定結果をリアルタイム表示
        └── 最終結果確定
              ├── 複数候補からベストマッチを選択
              └── 正誤判定
                    ├── 正解: 効果音 → 一定時間後に次の問題へ自動進行
                    └── 不正解: 効果音 → 「もう一度挑戦」ボタン表示
```

### 状態管理

| 状態 | 内容 |
|---|---|
| `hasStarted` | 練習が開始されたか |
| `isListening` | 音声認識中か |
| `interimText` | 認識中の暫定テキスト |
| `showFeedback` | フィードバック表示中か |
| `isCorrect` | 直前の回答が正解か |
| `recognizedNumber` | 認識した数字 |

## 4. 完了画面

**コンポーネント**: `features/card-sets/ui/card-set-flashcard.tsx` (内部状態で切り替え)

### 概要

カードセット内の全問題を完了したときに表示される画面。

### UI 構成

- 絵文字: 🎉
- タイトル: 「おめでとうございます!」
- メッセージ: 「全てのカードを完了しました!」
- **統計サマリー**: 正解数 / 不正解数 / 正解率
- **日別最短回答時間チャート** (`AnswerTimeChart`)
  - 棒グラフ (日別最短回答時間)
  - 破線 (全体平均ライン)
  - X軸: MM/DD 形式の日付
  - Y軸: 秒数
- **「別のカードセットを選ぶ」ボタン** → カードセット選択画面に戻る

## 5. 強制アップデートモーダル

**コンポーネント**: `features/force-update/ui/force-update-modal.tsx`

### 概要

アプリのバージョンがサポート最低バージョンを下回った場合に、全画面を覆うモーダルとして表示される。ユーザーは操作をブロックされ、アップデートを強制される。

### 表示条件

- アプリ起動時にリモートの `version-config.json` を取得
- 現在のアプリバージョン < `minimumVersion` の場合に表示
- ネットワークエラー時は表示しない（アップデートを強制しない）

### UI 構成

- 半透明ダークオーバーレイ (rgba(0,0,0,0.7))
- カード:
  - タイトル: 「アップデートが必要です」
  - メッセージ: 「このバージョンはサポートが終了しました。最新バージョンをインストールしてください。」
  - 「アップデートする」ボタン → App Store / Google Play を開く

### プラットフォーム別ストア URL

| プラットフォーム | URL |
|---|---|
| iOS | `iosStoreUrl` (version-config.json で設定) |
| Android | `androidStoreUrl` (version-config.json で設定) |

## ナビゲーション構成

```
RootLayout (_layout.tsx)
  ├── (tabs) - タブナビゲーター (タブバー非表示)
  │     ├── index (ホーム画面)
  │     │     └── CardSetFlashcard
  │     │           ├── CardSetSelector (カードセット未選択時)
  │     │           ├── 練習画面 (カードセット選択後)
  │     │           └── 完了画面 (全問正解後)
  │     └── explore (Explore 画面)
  ├── modal (モーダル画面)
  └── ForceUpdateModal (全画面オーバーレイ、条件付き表示)
```

## テーマ対応

全画面でライト / ダークモードに対応。`useColorScheme` フックと `useAppColors` フックにより、システムの設定に応じて配色が自動切替される。
